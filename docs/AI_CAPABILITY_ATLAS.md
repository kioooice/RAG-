# AI Capability Atlas

这是 `study` 工作区的 AI 系统能力地图。它不按课程、模型或框架排列，而按系统需要被验证的能力组织学习与实验。

## 角色分工

- 用户是控制面：定义目标、业务边界、方法选择、验收标准并负责纠偏。
- Codex 是执行面：负责环境、候选方案调研与集成、代码、实验、验证和证据整理。
- 共同决策顺序：成熟方案优先；必要时比较；先调整边界和规则；真实缺口出现后才自定义开发。

## 六大能力区域

| 能力区域 | 要回答的核心问题 | 典型验收证据 |
|---|---|---|
| 1. Document Intelligence | 多格式资料能否被可靠识别、解析、OCR、结构化并追溯来源？ | 格式覆盖、字段保真、表格/版面、页码定位、异常路由、人工复核率 |
| 2. 检索与知识访问 | 系统能否从正确范围找到支持问题的证据？ | Recall@K、MRR、延迟、失败样本、权限过滤、语料扩展稳定性 |
| 3. 生成与推理 | 系统能否基于证据完成回答、推理或工具调用，并在不确定时正确拒答？ | 引用正确性、事实一致性、任务成功率、拒答、成本和延迟 |
| 4. 评测与可观测性 | 能否持续发现能力变化、定位失败阶段并比较方案？ | 固定评测集、追踪记录、回归测试、成本/延迟、失败分类 |
| 5. 模型适配与优化 | 通用模型的真实缺口是否值得通过提示、微调、量化或蒸馏解决？ | 与通用模型对照、增益、数据质量、资源成本、泛化与回归 |
| 6. 系统集成与治理 | 能力能否安全、稳定地进入真实工作流并由人控制？ | 权限、隐私、安全、审计、可靠性、部署、回滚和人工接管 |
