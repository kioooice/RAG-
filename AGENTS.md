# AGENTS.md

## 项目定位

1. 本项目是个人技术学习实验，不是教学网站或公开产品。
2. 优先保留可观察、可修改的技术过程，核心技术不得完全封装成黑盒。
3. 不要隐藏 RAG 的解析、切块、Embedding、检索、重排序和生成过程。

## 环境与安全

4. Python 依赖只能使用项目根目录的 `.venv`，不得全局安装。
5. 系统级安装、管理员操作、删除已有文件和覆盖已有文件必须先获得确认。
6. 不得把密钥、Token 和密码写入代码、Notebook、配置或日志。
7. 长时间训练或大型下载前，先估算显存、内存、磁盘、时间和下载量；单项预计超过 1 GB 时必须暂停确认。
8. 不执行破坏性 Git 操作，不擅自连接远程仓库或推送。
9. 当前正式项目位于 D 盘 `study` 工作区；C 盘旧目录仅为迁移前副本，不得继续写入或删除。
10. 大型数据、模型和 Hugging Face、Torch、pip 缓存必须写入 `D:\AI-Lab` 下的对应目录。
11. 下载数据集或模型前必须报告预计大小；模型权重、数据集缓存、本地环境和密钥不得提交 Git。
12. Notebook 必须优先通过 `scripts\start_lab.ps1` 启动，项目脚本优先使用相对路径，不得写死用户目录。

## 修改与验证

13. 每次修改前说明目标，完成后说明修改文件和验证方式。
14. 每次实验只改变一个主要变量。
15. 实验必须记录数据版本、模型、参数、指标、运行时间和失败信息。
16. 不得使用测试集调参；训练集、验证集、测试集职责必须明确。
17. 错误记录需要保留，不能只留下最终成功结果。
18. 新增核心依赖时说明其用途，不得无理由堆叠框架。
19. 给用户的总结使用初学者可理解的中文，同时保留必要技术名词。

## 工作方式

- 先读取现有项目状态，再做最小、安全、可验证的修改。
- Notebook 中的关键输入、处理和输出应能直接观察。
- 完成实验后同时保存配置、指标和失败信息，不只保存最终图表。
